## Goal

Convert the uploaded Next.js 16 portfolio into a TanStack Start app that runs in Lovable's preview and can be published to Hostinger via a custom domain.

## Approach

Rebuild the app on top of Lovable's existing TanStack Start template. Keep all your design/content, replace Next-specific plumbing with TanStack equivalents, and move admin data + auth onto **Lovable Cloud** (so you don't need to bring your own Supabase credentials).

## What gets ported

**Pages → TanStack routes under `src/routes/`**
- `/` (home — hero, about, timeline, cases, testimonials, blog preview, certs, contact)
- `/about`, `/cases`, `/practice`, `/book`, `/media`
- `/blog` and `/blog/$slug`
- `/vlog`
- `/admin/login`, `/admin`, `/admin/posts`, `/admin/posts/new`, `/admin/posts/$id`, `/admin/vlogs`, `/admin/vlogs/new`, `/admin/vlogs/$id`

**Components (moved from `components/` → `src/components/`)**
All 16 components (Hero, Navbar, About, Timeline, Cases, Certifications, BlogPreview, Testimonials, Contact, Footer, Services, Process, FAQ, Videos, LanguageSwitcher, Providers) with these swaps:
- `next/image` → plain `<img>` (photos stay in `public/photos/`)
- `next/link` + `next/navigation` → `@tanstack/react-router` `Link` / `useNavigate`
- `next/font/google` (Inter + Cormorant Garamond) → `<link>` tag in `__root.tsx` head
- `"use client"` directives removed

**Assets**
`public/photos/*` preserved as-is. Favicons kept.

**Admin auth**
Rewired from custom JWT + `ADMIN_PASSWORD` to Lovable Cloud auth with a `user_roles` table (`admin` role) and a `has_role` RPC. Login page uses Supabase auth; admin routes gated by an `_authenticated` layout that checks the role.

**Data (blog posts, vlogs)**
Migration creates `posts` and `vlogs` tables with proper RLS: public can `SELECT` published rows, only admins can insert/update/delete via server functions.

**Removed / consolidated**
- `next.config.ts`, `next-env.d.ts`, `app/`, `postcss.config.mjs`, `tailwind.config.ts` (v3), `proxy.ts`, `eslint.config.mjs`
- `package.json` restored to Lovable's TanStack Start version + `framer-motion`, `lucide-react`, `jose` removed (unused after port)
- `lib/supabase.ts` replaced by `src/integrations/supabase/client.ts` (auto-generated by Lovable Cloud)

## Hosting on Hostinger

Lovable itself hosts the published site on a `*.lovable.app` URL. To use your Hostinger-purchased domain:

1. Publish the project in Lovable (I'll do this once the port builds cleanly).
2. In Hostinger's DNS panel add:
   - `A` record `@` → `185.158.133.1`
   - `A` record `www` → `185.158.133.1`
   - `TXT` record `_lovable` → value Lovable shows during setup
3. Connect the domain in Lovable → Project Settings → Domains (both `yourdomain.com` and `www.yourdomain.com`).
4. SSL auto-provisions within ~72h (usually much faster).

Hostinger is only the domain registrar in this setup — Lovable serves the site. Your Lovable-hosted app is what your domain will point to.
